## 1. 划重点

- 中文界面里，不能使用`英文 slash 命令`过滤。
	- 先输入 `/` 打开`命令面板`，再用`中文关键词`过滤。
	- 比如输入`“侧”`，就能找到`侧边`对话。
- 记，没有任何意义，每次输入 `/` 自己选就好了，如图
	- ![202520260508174653386.webp|480](/assets/os/202520260508174653386.webp)

## 2. Codex 自带且个人常用的

- `命令面板`：
	- 忘了命令就先输入 `/`，再用中文关键词搜。
- `状态`：
	- 查看模型使用情况限额情况
- `计划`：
	- 大任务先拆方案，确认后再实施。
- `压缩`：
	- 长对话后压缩历史，保留关键结论。
- `侧边`：
	- 临时问一句，不污染主线。

## 3. 我自己的项目命令

这些不是 Codex 全局命令，而是我在 `~/832` 里给 Agent 准备的项目命令。

- `/new-note`：
	- 把材料整理成 `os/notes` 里的长期 note。
- `/new-issues`：
	- 把当前讨论整理成 `GitHub Issue`。
- `/get_next_index`：
	- 给 `notes` 里的待编号文件分配下一个全局编号

## 4. 命令的价值

- `命令`的价值，是让我随时能控制 `Codex`。
	- 需要把材料变成长期 note，就用 `/new-note`。
	- 是否公开发布，不靠目录分类，靠文件名里的 `@832`、`@ing` 等标记。
	- 需要留下一个待处理问题，就用 `/new-issues`。
