## 1. 划重点

- 中文界面里，不能使用`英文 slash 命令`过滤。
	- 先输入 `/` 打开`命令面板`，再用`中文关键词`过滤。
	- 比如输入`“侧”`，就能找到`侧边`对话。
- 记，没有任何意义，每次输入 `/` 自己选就好了，如图
	- ![202520260508174653386.png|480](/assets/os/2026/05/202520260508174653386.png)

## 2. Codex 自带且个人常用的

- `命令面板`：
	- 忘了命令就先输入 `/`，再用中文关键词搜。
- `状态`：
	- 查看模型使用情况限额情况
- `计划`：
	- 大任务先拆方案，确认后再实施。
- `压缩`：
	- 长对话后压缩历史，保留关键结论。
- `侧边`：
	- 临时问一句，不污染主线。

## 3. 我自己的项目命令

这些不是 Codex 全局命令，而是我在 `~/832` 里给 Agent 准备的项目命令。

- `/new-blog`：
	- 把材料整理成待审公开文章。
- `/new-corpus`：
	- 把材料整理成对内可复用内容。
- `/new-embodied`：
	- 把讨论提炼成原则、方法或复盘。
- `/new-issues`：
	- 把当前讨论整理成 `GitHub Issue`。
- `/get_next_index`：
	- 给 `corpus`、`blog`、`embodied` 里的待编号文件分配下一个全局编号

## 4. 命令的价值

- `命令`的价值，是让我随时能控制 `Codex`。
	- 最近的对话可以总结成博客，就用 `/new-blog`。
	- 一段材料只适合对内复用，就用 `/new-corpus`。
	- 一个判断已经能指导行动，就用 `/new-embodied`。
	- 需要留下一个待处理问题，就用 `/new-issues`。
