## 划重点

- `语雀` 可以先当一个富文本语料中转站。
	- 从企微文档、飞书文档、腾讯文档这类富文本工具复制内容到语雀，格式基本能保住。
	- 图片也能被语雀接住，不只是纯文本。
- 有了 `语雀 MCP` 以后，语雀里的内容就更容易被 Agent 读到、搜到、连接起来。
	- 这比直接处理`腾讯文档`或者`飞书文档`顺手很多。
	- 因为我还 2 年的`超级会员`
- 我现在真正的痛点不是没有文档，而是文档在腾讯文档里。
	- 产品写 PRD 都在那里。
	- 语料很多，但对 AI 来说不好拿这个语料

## 一个发现

语雀它能比较完整地吃下富文本（包括图片、表格等）内容，然后又能被 MCP 读写。

>  老实说，语雀我认为一直是一个很好的产品，可惜了

## 怎么用

- 产品文档仍然在腾讯文档或者飞书里协作
- 需要进入 AI 语料的 PRD，手动复制到语雀
- 语雀作为富文本中转层。
- Agent 通过 `语雀 MCP` 读取、总结、关联。

## 最后，吐槽下

- 企微文档太垃圾，导出啥都不行，各种混乱格式，MCP 功能的话要各种新建应用各种流程
- 飞书功能是比较多，但是太眼花缭乱了
