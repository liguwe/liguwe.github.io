## 1. 划重点

- 最近频繁用 `语雀文档` 和 `MasterGo` 开发需求，就是一个很小但很准的例子。
	- `语雀` 放我个人的提示词
		- 其实 PRD 管理很混乱，可能在腾讯文档上，也可能在飞书文档上，总之很乱
		- 但发现通过语雀，一键复制能够完美还原其他平台的文档
	- `MasterGo` 放设计稿
- 这些在 `Codex` 中被 `MCP` 接起来以后，研发效率提升的地方不是少敲几个字，而是少了很多`上下文搬运和转述误差`。

## 2. 小事

- 用 Agent 最耗时间的是，让它知道“到底要写什么”。
- 为此，我们能要做
	- 需求文档要复制
	- 设计稿要截图
	- 字段含义要解释
	- 页面交互要描述。

> 所以每转述一次，信息就薄一层；每截图一次，细节就丢一块。最后 AI 其实花了大量时间是在`猜上下文` 

所以`MCP` 就是能接触到`更完整的现场`。

## 3. MCP 的意义

当下，`MCP` 对我来说，最大的意义它减少的不是输入成本，而是协作里的**上下文损耗**。

## 4. 更大的判断

除了模型能力的持续提升，以前普通工具之间能被可靠地接起来，这一点也很重要

## 5. 更多

其实，也许这一切都很 `虚无` ，我总是想到了 `《三体》里人类发现三体人真实能力的那一刻的画面`
