## 1. 划重点

- 首先，这是`个人`的，所以，没必要使用 `Discussions` ，徒增复杂度
- 再者，不是 `Linear` ，也不是 `Notion`
	- `Projects`就足够自己使用
	- 另外，现实的因素，即`钱`的因素
- 最后，怎用 `GitHub Projects` 管理 `issues` ，即一个`多维表格`
	- 新建`Issues Dashboard` ，把分散在 `GitHub Issues` 里的事，放到一个能全量查看、能按状态推进的地方。

## 2. 个人创建的所有 issues

- 入口是 [https://github.com/issues](https://github.com/issues)。
	- 最近入口是 [https://github.com/issues/recent?page=1](https://github.com/issues/recent?page=1)。

> 因为 issues 可能分散到不同的项目中，通过这个页面来管理

## 3. 再用 Project 做 `工作台` 或者 看板

- 个人 Projects 入口是：
	-  [https://github.com/liguwe?tab=projects](https://github.com/liguwe?tab=projects)。
- Issues Dashboard： https://github.com/users/liguwe/projects/7/views/2

## 4. Project 里不一定能按 Issue 时间排序

不能设置，因为导入到看板时，没有这个字段，但是也不是强诉求，对于个人来说，需要就到 
- 最近入口是 [https://github.com/issues/recent?page=1](https://github.com/issues/recent?page=1) 就好了
