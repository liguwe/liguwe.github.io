## 1. 核心观点

- 人要训练自己，让万事万物穿过自己。
	- 穿得过去：
		- 会变成经验。
		- 会变成直觉。
		- 会变成判断。
		- 会变成生命力。
	- 穿不过去：
		- 会变成认知的房子。
		- 会变成情感的房子。
		- 房子一开始是保护，后来可能变成遮蔽。
- 孟岩把几件事讲成了一件事：
	- 创作。
	- 投资。
	- 网球。
	- 身体。
	- 过去的伤害。
	- 臣服。
- 这一期对我的提醒：
	- 准备和上场是两种状态。
	- 准备阶段要高密度输入、训练、补事实。
	- 上场阶段要允许现场发生，不能被提纲绑死。
	- 焦虑时，我经常想把路径提前规划完整，但真正有生命力的东西往往是在现场长出来的。

## 2. 大纲

### 2.1. 创作最让人兴奋的地方，是不知道会发生什么

- 孟岩说，每次走进录音间最兴奋的，不是已经知道要聊什么，而是不知道会发生什么。
- 他录播客前会做很多准备：
	- 读嘉宾专访。
	- 看嘉宾写的东西。
	- 听播客和直播。
	- 用第一手材料理解这个人。
- 真正开始录时，他不拿逐字稿，也没有具体提纲。
	- 只有一个大概方向。
	- 相信嘉宾。
	- 相信自己。
	- 进入一场聊天。
- 对我自己的提醒：
	- 准备不是为了把现场控制住。
	- 准备是为了让自己有足够材料进入现场。
	- 如果准备最后变成控制欲，就会堵住真正的东西。

### 2.2. 好内容是生长出来的

- “好的内容是生长出来的，而不是计划出来的。”
- 这句话容易被误解成不需要准备。
	- 实际上它说的是：
		- 准备很重要，但内容真正出现时，不是从提纲里执行出来的。
	- 它是材料、环境、身体、对方反应和当下情绪一起穿过自己之后长出来的。
- 这也适用于 Agent 工作：
	- 模型、工具、上下文、命令、流程都重要。
	- **真正能跑起来的工作流，通常是在真实任务里长出来的**。
	- 看真实错误。
	- 理解真实边界。
	- 保留状态。
	- 修脚本。
	- 下一次复用。
- 警惕：
	- 太早把流程写成漂亮规则，可能只是另一座“认知的房子”。
	- 看起来完整，实际挡住了现场。

### 2.3. 直觉不是拍脑袋

- 孟岩讲直觉时，不是在贬低逻辑。
- 直觉和逻辑只是不同出口。
	- 逻辑解决已知信息里的推演。
	- 直觉有时补上模型之外的信息。
- 直觉来自：
	- 身体。
	- 经验。
	- 信息。
	- 情绪。
	- 长期训练。
- 关键不是选逻辑还是选直觉。
- 关键是把自己变得足够好：
	- 足够丰富。
	- 足够松弛。
	- 足够有经验。
	- 足够能听见身体给出的答案。

### 2.4. 认知的房子

- 人为了理解世界，会不断建模。
	- 看到四四方方、带轮子的东西，不用每次重新分析，就知道是汽车。
	- 这节省脑力，也带来掌控感。
- 问题是模型会变成锤子。
	- 手里有锤子，看什么都像钉子。
	- 人会用旧模型解释过去。
	- 用旧模型预测未来。
	- 给不符合模型的东西贴标签。
	- 最后误以为是世界出了问题。
- 对我自己的提醒：
	- **很多“我已经懂了”，只是模型太顺了**。
	- 很多“这个方向不对”，可能只是它不符合旧房子。
	- 真正要看的是：
		- 这个模型还能不能解释新的事实。

### 2.5. 情感的房子

- 情感的房子比认知的房子更隐蔽。
- 孟岩举了开车时看到广告牌的例子：
	- 路边的树、建筑、车、交警，本来都能穿过自己。
	- 看到一个广告牌，**勾起不好的记忆**。
	- 人突然被卡住。
	- 后面继续出现的车、树、建筑和天空，其实都没有真正看见。
- 这对我很有用：
	- 很多时候，**人看起来是在处理当下的事，其实是在对过去反应**。
	- 看起来是某个任务让人烦。
	- 看起来是某个错误让人焦虑。
	- 看起来是某个评价让人失控。
	- **底层可能是旧的房子被撞到了**。
- 房子一旦被撞到：
	- 后面的信息进不来。
	- 新的判断出不去。
	- 人会用很大的力气证明自己在解决问题。
	- 实际上可能只是在守住那座旧房子。

### 2.6. 投资是认知的变现

- 孟岩用“认知的房子”解释前 9 期嘉宾的投资方式。
- 每个人都在自己的房子里筛选信息、承受风险、做出选择。
- 例子：
	- 南天相信规模效应、把手弄脏、不断进化。
	- 嘉禾相信世界不可预测，所以用分散和低估保护自己。
	- 肖宇重视自己和公司的长期关系。
	- 方丈在更长时间维度前保持敬畏。
	- 海燕在问题很多的世界里仍然能扣动扳机。
	- 简七选择简单的大类资产配置。
- 这里的重点：
	- 投资不是知识竞赛。
	- **投资是一个人到底用什么模型筛选世界**。
	- **同样的信息穿过不同的人，会变成不同选择**。

### 2.7. 网球也是同一件事

- 击球前可以拆动作：
	- 盯球。
	- 跑到落点。
	- 左手瞄球。
	- 引拍。
	- 蹬腿。
	- 转体。
	- 随挥。
- 真正击球时，如果脑子里同时想着这些动作，身体会僵。
- 球拍接触球只有零点几秒。
	- 想太多，动作就散。
	- 正确的做法是训练时拆动作，上场时让身体工作。
- 这和创作、投资、写代码都像：
	- 平时建立模型。
	- 现场相信身体。
	- 场下训练。
	- 场上放手。

### 2.8. “算了”不是轻飘飘的放下

- 最重的一段，是孟岩说离开且慢两年后的答案是“算了”。
- 这个“算了”不是假装不痛。
	- 阴影还在。
	- 看到相关名字依然会难过。
	- 依然会短暂卡住。
- 但如果回到过去，他不愿意改变那些经历。
	- 因为那些经历把他推到了今天的位置。
	- 让他知道什么对自己真正有价值。
	- 让他能把想做的东西注入公司、产品和伙伴。
- 这段比“感谢苦难”真实。
	- 伤害就是伤害。
	- 房子还在。
	- 卡住也是真的。
	- **但人可以慢慢重新理解那段经历在生命里的位置**。

### 2.9. 臣服不是躺平

- 孟岩特别强调：
	- 臣服不是佛系。
	- 也不是躺平。
- 臣服更像是：
	- 承认自己的认知和情感房子都很小。
	- 承认自己看不全生活。
	- 放弃对结果的过度控制。
	- 仍然付出全部力量，走向眼前那条不确定的路。
- 它要求的**不是更少行动**。
- 它要求的是**更少自我纠缠**。

## 3. 有意思的观点、原话和非共识点

- 每次进录音间让他兴奋的，是不知道会发生什么。
	- 这和很多人理解的“专业”相反。
	- 很多人以为专业就是掌控更多。
	- 这里的专业更像是：
		- 准备足够多，然后有能力进入未知。
- `not doing list` 用在播客里很有意思。
	- 不要总想着自己表达。
	- 不要想着自己的话是不是高级。
	- 不要想着展现自己。
	- 不要在嘉宾回答时想着下一个问题。
	- 真正重要的是对对方产生好奇。
- BGM 那段也能看出创作者边界。
	- 有人觉得前奏太长，他说这是自己的表达方式，可以不听。
	- 这不是任性。
	- 这是保留自己的节奏。
	- 世界一直要求人节省时间、提高效率、不要浪费，但有些东西只有在不赶时间时才会出现。
- 情感房子的例子很实用。
	- 一个广告牌就能让人卡住。
	- 后面的世界还在发生，但人已经不在场。
	- 这解释了很多焦虑、逃避和反复反应。
- “算了”这一段最真实。
	- 它不是原谅所有事。
	- 也不是把伤害说成好事。
	- 它只是承认：
		- 过去的事还会卡住我，但我不再用它决定我接下来怎么活。

## 4. 金句

- 内容和给女士的惊喜一样，都是在一个预期之外的时间，提供一个预期之外的惊喜。
- 好的东西是生长出来的，而不是计划出来的。
- 我们应该**让写作来完成写作，让播客来完成播客，让产品来完成产品**。
- 生命并不长，不要再赶时间了。
- 我们只能看见外面，**但几乎所有的事情都发生在我们的内心**。
- 人非常重要的一个课题，是知自己，做自己，知道自己是谁，把自己变得更好，然后 100% 地把自己表达出来。
- 当你遇到难关，或者有人找你麻烦时，可以把这个人想象成一个能预知未来、而且爱你的人。他希望你成就不凡，所以必须把你推向一个你当下不认同的方向。
- 你的每一段经历，其实都是为了你的下一步做好了铺垫，只是你不知道下一步是怎么回事。
- 未来无人知晓，但我愿意放手，愿意纵身一跃，愿意去冒险，愿意相信自己的身体和心。

## 5. 启示

- 对写作：
	- 不要把提纲当成果。
	- 先让材料穿过自己。
	- 写出来以后再整理结构。
- 对代码和 Agent：
	- 不要在还没看到真实错误前，把流程设计得过度完整。
	- 真实任务会暴露边界。
	- **能跑、能复用、能验收，比漂亮说明更重要**。
- 对焦虑：
	- 焦虑时先看是不是旧房子被撞到了。
	- 不要急着再搭一套工具、再列一套计划。
	- 先把眼前发生的事情看清楚。
- 对长期行动：
	- 场下训练。
	- 场上放手。
	- 让事情穿过自己。
	- 然后给出反应。
