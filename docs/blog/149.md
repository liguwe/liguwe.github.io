---
title: "149. 关于 CLI：它是 Agent 的底层接口，不是人的"
---

## 重点

- `CLI` 的价值在于给 Agent、脚本和自动化留一个稳定入口。
- 我真正反感的是把 `CLI` 当成人和 Agent 协作的唯一界面。
- 在 Agent 时代，`CLI` 更像底层接口形态：可组合、可脚本化、可被别的系统调用。
- 读 `Codex CLI`、`opencode`、`gemini-cli`，是为了拆清 Agent 系统的最小可执行剖面。
- 个人结论很清楚：`CLI` 继续保留为底层能力、学习入口和自动化接口，日常协作界面交给更适合人的容器。

## CLI 不是界面偏好

以前我讨厌 `Claude Code`，很大一部分原因就是讨厌 `CLI`。

这不是矫情。纯终端交互确实有成本：

- 上下文不容易看。
- 文件和 diff 不如编辑器直观。
- 多线程、多任务、多材料切换很累。
- 很多时候，人只是想在正在写代码的地方叫 Agent 一下，而不是跳到另一个黑框里。

所以问题不该写成：

```text
我到底喜不喜欢 CLI？
```

更准确的问题是：

```text
CLI 在整套 Agent 工作流里，应该站在哪一层？
```

`CLI` 不是审美选择。它是一种接口形态。

## 为什么 Agent 时代更需要 CLI

`CLI` 最重要的特征是天然适合被别的东西调用。

- 有稳定命令名。
- 有参数。
- 有 stdin / stdout / stderr。
- 有 exit code。
- 能放进脚本。
- 能被 CI、Raycast、Codex、Cursor、Zed Agent Panel 或其它 Agent harness 包起来。

这也是为什么很多 SaaS、IM、办公平台现在都开始补 `CLI` 和 `MCP`。

过去软件主要给人点按钮。Agent 不适合这样用软件。Agent 需要稳定入口：

- `MCP` 负责把上下文接出来。
- `CLI` 负责把能力暴露成可执行动作。

飞书这个例子很小，但足够说明问题。文档、妙记、音频转写本来就在飞书里；接上 `飞书 CLI` 和 `飞书 MCP` 以后，Codex 才能把这些能力串成一条素材链路。

这件事的重点是软件能力开始变成 Agent 可以调用的接口。

## CLI 是理解 Agent 系统的最小剖面

`general-agent-lab` 现在主读 `Codex CLI`，用 `opencode`、`gemini-cli` 做对照。

目的很明确：用最薄的一层把 Agent 系统骨架拆出来。

`CLI` 是理解 Agent 系统的一条窄切口：

- 入口在哪里。
- 命令怎么分发。
- 会话怎么创建。
- 上下文怎么装载。
- 模型客户端怎么接进 Agent Loop。
- 工具怎么注册和调用。
- 权限、沙箱、approval 怎么落地。
- diff、日志、测试结果怎么反馈给用户。

相比一上来研究完整 GUI，`CLI` 更薄，路径更短，更容易看清骨架。

比如 `Codex` 和 `opencode` 的 npm 入口都是 Node shim 包原生二进制：npm 负责分发和命令暴露，真正的 Agent 能力在后面的二进制里跑。

这件事对我有启发：`CLI` 的核心价值经常不在界面层，而在它把一整套复杂能力压成一个可安装、可运行、可组合的入口。

## CLI 的负担

不能神化 `CLI`。

它对人不总是友好。

- 长上下文阅读不友好。
- 多文件编辑不友好。
- 视觉反馈不够。
- diff、引用、图片、设计稿、文档现场都容易变薄。
- 人和 Agent 来回确认时，终端不是最舒服的地方。

这也是我前段时间一直在卸载、收敛、告诫自己的原因。

反对的是工具空转，不是反对 `CLI` 本身。

真正的问题是：当 `CLI` 变成新的折腾对象，它就不再服务产出，而是在制造一个体面且复杂的避难所。

所以 `CLI` 不能被放在人的唯一工作界面上。

## 我的分工判断

`CLI` 继续存在，而且会更重要。

但它的位置要摆对：

- 底层能力：把复杂 Agent 能力变成稳定命令。
- 学习入口：用最短路径拆清 Agent harness 的骨架。
- 自动化接口：让脚本、CI、Raycast、Codex、Cursor、Zed 这些上层容器能调用它。
- 工具层：给 Agent 用，而不是强迫人天天用。

人的日常界面应该更接近编辑器、桌面端、任务面板或其它更能保留现场的容器。

所以结论很简单：**CLI 仍然是 Agent 的底层接口，但不该再被当成人类协作的唯一界面。**

## 留给 ACP 的问题

如果 `CLI` 是底层能力，那日常协作界面应该长什么样？

最近 Zed Agent Panel 接 `claude-code`、`codex-cli`、`cursor-cli`、`gemini-cli` 这件事，给了一个更具体的答案：

```text
让 CLI Agent 继续做底层，让 ACP 把它们接回编辑器现场。
```

这就是下一篇要写的问题。
