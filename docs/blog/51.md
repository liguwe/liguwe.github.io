---
title: "使用 vercel-labs skills 管理个人 AI 技能"
---

先说结论：

`vercel-labs/skills` 是安装器，不是技能来源。

我的技能来源是：

```text
/Users/liguwe/832/skills/skills
```

> remote 维护到 Github 仓库中

我用 `npx skills add` 把这里的技能分发给 Codex、Claude Code、Cursor。

## 1. 我现在的分工

现在这件事分成两层：

```text
/Users/liguwe/832/skills        # 我的个人 AI 配置源头
/Users/liguwe/832/skills/skills # 真正可安装的 skills
npx skills add                  # vercel-labs/skills 提供的安装器
```

所以不要混：

- 技能内容：我自己维护。
- 安装工具：`vercel-labs/skills`。
- 使用方：Codex、Claude Code、Cursor。

## 2. 为什么这样用

因为三个工具的目录不一样。

如果每个工具里都手动维护一份 skill，很快就会分叉。今天改 Codex，明天忘了 Claude Code，后天 Cursor 还是旧版本。

现在我只改源头：

```text
/Users/liguwe/832/skills/skills/personal-blog-writing/SKILL.md
```

然后用安装器同步出去。

## 3. 当前安装命令

全量安装：

```bash
npx skills add /Users/liguwe/832/skills/skills \
  -g \
  -a codex \
  -a claude-code \
  -a cursor \
  --skill '*' \
  -y
```

检查有哪些 skill：

```bash
npx skills add /Users/liguwe/832/skills/skills --list
```

这两个命令基本够用了。

## 4. 现在装了哪些

当前有 5 个：

- `personal-blog-writing`
- `png-optimize`
- `qr-code-generate`
- `translate-docs-to-zh-vitepress`
- `xiaoyuzhou-audio-download`

其中 `personal-blog-writing` 是最新加的，用来约束我写博客：写给自己看，少铺垫，结论直接。

## 5. 我需要记住的边界

`skills/` 只放真正的 skill。

我现在把仓库外层扩成了个人 AI 配置源头：

```text
skills/    # 可安装技能
rules/     # 规则源稿
commands/  # 命令源稿
mcp/       # MCP 模板
```

但只有 `skills/` 会走 `npx skills add`。

`rules/commands/mcp` 只是源稿，后面要按工具适配。

比如项目级命令还是放项目里：

```text
.cursor/commands/
.claude/commands/
.agents/skills/
```

不要为了统一，把所有东西都塞进 skill。

## 6. 最后记一句

我的模型是：

```text
一个源头，多个适配。
```

源头在 `/Users/liguwe/832/skills`。

安装器用 `vercel-labs/skills`。

真正运行时，还是尊重 Codex、Claude Code、Cursor 各自的入口。
